Typescript 使用 infer 的 4 种场景

最近看到了两位大佬天花板级别的类型体操：

  • 用 TypeScript 类型运算实现一个中国象棋程序 - 徐飞
  • TypeScript 类型体操天花板，用类型运算写一个 Lisp 解释器 - brambles

看完只能献上我的 🧎‍♂️。网上也出现一些 type-challenges 上的面试骚题，大部分是考察 infer 的应用。

实际项目中，抽象不同业务的相同流程时，我有使用 infer 推断具体业务类型。infer 的强大在于可以通过模式匹配可以从泛型中提取到类型，总结有 4 种场景：

  • 对象属性类型
  • 函数参数类型和返回值类型 Function Parameters and Return type
  • 数组 Array
  • 模板字符串 Template Literals

使用前注意：infer 只能出现 extends 条件类型1 语句中，且类型变量只能在 true 分支中使用。

条件类型语法有点像 JavaScript 的三元运算 (condition ? trueExpression : falseExpression) ：

1

SomeType extends OtherType ? TrueType : FalseType;

对象属性

condition 的写法是属性作为 key，值使用 infer ：{ prop : infer V }。下面代码从通用的接口类型中获取 todo 数据类型

interface ApiResponse<T> {
  code: number
  data: T
}
interface Todo {
  content: string
  completed: boolean
}
type TodosResp = ApiResponse<Todo[]>

type GetRespData<T> = T extends { data: infer V } ? V : never
type DataInTodoResp = GetRespData<TodosResp> // Todo[]

函数参数和返回值

写出函数类型声明 (...args: any[]) => any 然后填空即可

返回值类型

1

type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never

参数类型

type MyParameters<T extends (...args: any[]) => any> = T extends (
  ...any: infer P
) => any
  ? P
  : never

数组

一类仍然是模式匹配，通过解构赋值2，取数组第一个/最后一个的类型

1
2

type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never

一类要使用递归，在类型系统中实现 JavaScript Array.includes 函数，解构取到第一个元素和由剩余元素组成的数组，第一个匹配成功返回 true，未匹配成功继续用剩下的数组匹配。

type Includes<T extends readonly any[], U> =
		T extends [infer F,...infer R]
			? (Equal<F,U> extends true ? true : Includes<R,U>)
			: false

模板字符串

删除两段的空白字符串 Trim，解法是模板模式配合递归

1
2
3
4
5
6

type Chars = ' ' | '\n' | '\t'
type Trim<S extends string> = S extends `${Chars}${infer SS}`
  ? Trim<SS>
  : S extends `${infer SS}${Chars}`
  ? Trim<SS>
  : S

总结

4 个场景捋下来，使用 infer 提取类型的关键在于写出符合不同数据结构的模式，然后填空即可。日常业务开发，这些袋鼠摇基本够用了~ 🦘
