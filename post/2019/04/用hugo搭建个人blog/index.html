以往写博客没坚持下来，除了主观原因外，客观上也有工具不太给力、更新维护不方便的原因。一直有看到在github上维护自己博客的，今天也整一下。

博客更多是自己的总结，写作是第一位；美观是写作的动力，也是更好的阅读体验；文章写得多，检索分类总是少不了；还有读者互动、多平台发布。

jekyll、hexo、hugo怎么选？

  • jekyll 使用 ruby 编写
    • 优点：github官方集成，老牌工具生态主题多
    • 缺点：构建较慢
  • hexo 使用 nodejs 编写
    • 优点：nodejs 对前端友好，主题较多，构建速度介于 hugo 和 jekyll 之间
    • 缺点：没有喜欢的主题，看到 node_module 就头大
  • hugo 使用 go 编写
    • 优点：构建速度快
    • 缺点：主题少

hugo性能好、有喜欢的主题、对go也有兴趣，就选它了，按照官方教程把网站搭好。现在网站还很丑，以后有时间再优化，先完成今天的目标。
